Background:

We need call apple API to get available iOS update information: https://gdmf.apple.com/v2/pmv

If issue request with default http client, there would be SSL Handshake error like below:

Since the apple API server’s certificate is signed by itself called Apple Server Authentication CA, not the trust CA like Digicert or GlobalSign in the internet.

So, for such scenario, it will be necessary for us to add this API server’s certificate to our xm-server tomcat truststore, make sure that it would be fine when issue request from our server side.

​ Hello everyone, this video is to show new feature -required minimu version.

With iOS 17, MDMs can now enforce a minimum operating system version on enrolling devices when using Automated Device Enrollment/Device Enrollment Program.

Required Minimum Version Key Areas including few components

•Allow old devices to enroll: If enabled, devices can enroll even if they could not upgrade to the current required minimum version. It is enabled by default

•Specified version option: Turn on specified version option, admin could input specified version manually. This option is closed by default.

•Specified version: It’s one text field for admin to input maunally. If the device has an iOS version lower than this, it will kick off update process on device. If the version is expired in the future, a minimum version of the available version list is used. Please enter the correct version number, otherwise unknown errors may occur.

•Available iOS versions:It’s one combobox component for admin to select version they want. If the device has an iOS version lower than this, it will kick off update process on device. If the version is expired in the future, a minimum version of the available version list is used. If set as None, it will not take effect.

PS : The iOS version list is updated by CEM through periodic requests to Apple’s API.

Let’s see demo video here.

If we edit or add dep account, and go to iOS settings page, scroll this page and there would be one new area for this feature.Admin could iput version manually or select version.

Let’s take a look at the device enrollment process.

The devices system version is detected to be below the required minimum version set by the admin which is 17.1.

In this case, an update page will be displayed prompting the user to update the device.

After the update is completed, the device will restart and re enter the enrollment process.

At this point, the devices system version will meet the requirements allowing for the normal user authentication process.

The device can be activated and enrolled to CEM.Once the device is activated, it can be observed that the system version has been upgraded to 17.1.

Additionally, the MAM enrollment is completed in the secure hub. Simultaneously in the CEM, The displayed system version for the device is 17.1.

Thanks everyone, that’s all.
