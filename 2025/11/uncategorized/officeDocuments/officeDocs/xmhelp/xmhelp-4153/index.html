Mamnetworksdk will reload or start request anew when cross-site issues occur. Generally, this behaviour is for enhanced security, but in this customer’s scene, it will cause failed navigation.

Phil:

Hi Junkang, since iOS 12, Apple has relaunched the WKWebView process on cross-navigations. If we don’t do the reload/restart, many pages will not render at all, so it can’t be disabled. Does the customer have the problem regardless of whether didRelaunchProcess does a reload or starts the request anew? There is the feature flag mentioned in that powerpoint slide to control that behavior on a per-customer basis.
