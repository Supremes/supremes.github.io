Guide - How to renew CA group with CEM public API

Step 1 - Get auth_token with cloud administrator account

  1. Replace #cloud_admin_user_name# with your cloud admin user name
  2. Replace #cloud_admin_password# with your cloud admin password

After execution, you will get the auth_token.

Step 2 - Generate new CA group

  1. Replace #auth_token# with the auth_token you get from Step 1.
  2. Replace #new_ca_group_name# with your prefered name, such as “ca_2022”.
  3. Replace #root_ca_password# with the new root CA cert private key password.
  4. Replace #server_ca_password# with the new server CA cert private key password.
  5. Replace #devices_ca_password# with the new devices CA cert private key password.

You can make the 3 password the same one if you wish.

After execution, you should have already created a new CA group now, but haven’t issue yet.

Step 3 - Query CA group

  1. Replace #auth_token# with the auth_token you get from Step 1.

After execution, you should see the information about new CA group with old CA group together, but the new CA group currentIssuingCA is false.

Step 4 - Issue the new CA group

  1. Replace #auth_token# with the auth_token you get from Step 1.
  2. Replace #new_ca_group_name# with the one you given in Step 2.

After execution, the new CA group should be issued, you can re-do Step 3 to check the status of new CA group currentIssuingCA, it should be true.

Summary

All those commands should be run in a bash like terminal, and you need to have curl installed.

After all those commands executed, the CA group is renewed already, and you can have your decision on whether to delete the old one. Please refer to the CEM public API documentation.

Keep old CA group won’t affect anything, so you can just leave it here.

If you prefer to use some other GUI tool to send HTTP request, such as Postman, that will also work. You can config the parameter in the Postman regarding to the curl command.
