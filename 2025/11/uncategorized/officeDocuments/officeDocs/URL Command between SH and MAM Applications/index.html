MAM SDK Usage

kMDXREQUEST_SDK_APP_LOGON

Params

  • withReauth： YES

Scenario

For MAM App call.

+handleSAMLTokenRequestFailureWithTimeout:andCompletionBlock:

+configManagerAppCoreInit

FTU

+performLogon

This API provides an app developer a way to invoke a Secure Hub login from a delegate called back from the MAM SDK

getSTATicketWithCompletionBlock:performLogonOnFailure:

Get STA ticket failed if auth failed but not cert failure, AG session is unvalid, flip to SH to renew AG session.

renew ag session:

FTU Scenario: SH -> Gateway(load balance) -> CEM

Other Scenario(STA unvalid): Appcore -> Gateway(load balance) -> CEM

detectAppUpgradeAndLogon

This app upgraded, need to sign on in Secure Hub to fetch updated app info.

showNowLogonAlertForUnexpectedSHData

Alert: Managed Data Changed

The app was already managed, but couldn’t decrypt managed containers. Need to perform logon

SH Logic

Perform MAM Login and Construct Payload containing application data, including:

kCTXSSO_APP_INFO:

  • AppResourceID
  • AppResourceURL
  • AppResourceURLTranslated
  • AppLogLevel
  • AppLogTarget

kCTXSSO_SUPPORTS_MAMAPPDATA_SHARING

  • supportsMAMAppDataSharing

UNMANAGED_CONTAINER_KEYCHAINACCESSGROUP

  • unmanaged container
